## 🎥 Demo Script (Interview / Code Review 用 5 分デモ)

面接やコードレビューの場で、DocuFlow を **5 分で紹介するための台本** です。  
画面遷移と話すポイントをあらかじめ決めておくことで、「どこを見てほしいか」が明確になります。

---

## 0. 前提

- 事前にブラウザで `https://docuflow-azure.vercel.app` を開いておく
- サインイン済みの状態にしておく（もしくはログインから始める）

---

## 1. ログイン → ダッシュボード（約 1 分）

1. `/auth/login` を開く
   - 「メール & パスワード / Google ログインの両方に対応している」ことを一言で説明  
   - パスワード強度チェックやエラー表示など、基本的な UX に触れる
2. ログインして `/app` に遷移
   - 「ここが DocuFlow のメインダッシュボードです」と宣言
   - 画面上部のインサイトカード（ドキュメント数、ピン、お気に入り、直近 30 日、カテゴリなど）をざっと見せる  
   - ついでにヘッダー右上の `What's New` / 通知ベル / ユーザーメニューに軽く触れる

話すポイント例:

- 「AI 要約済みドキュメントを中心に、“探す時間” を短縮するダッシュボードになっています」
- 「右上には通知ベルや What's New、ユーザーメニューがあり、SaaS としての基本要素を揃えています」

---

## 2. ドキュメントアップロード → AI 要約 / 検索（約 2 分）

1. `/app` 下部のアップロードエリア（Drag & Drop）を見せる
   - サンプル PDF / Word を 1 つアップロードしてみる
   - アップロード後に自動で生成される「タイトル / 要約 / タグ」を見せる
2. 検索バーでキーワード検索
   - 例: 「認証」「売上」など、登録済みサンプルドキュメントがヒットしやすいワード
3. ドキュメント詳細ページ（`/documents/[id]`）に遷移
   - AI 要約 / タグ / バージョン履歴 / コメントをざっとスクロール

話すポイント例:

- 「AI 要約・タグ付け・タイトル生成までを自動で行うことで、“アップロードしておけば後からすぐ探せる” 状態を作っています」
- 「検索は単なるキーワード検索ではなく、ベクトル検索も組み合わせて、意味ベースで関連ドキュメントを返すようにしています」

---

## 3. コメント & 通知 / 組織・RBAC（約 1 分）

1. ドキュメント詳細でコメントを 1 件追加
   - メンション（`@`）を軽く示してもよい
2. `/app` に戻り、ヘッダーの通知ベルを開く
   - 先ほどのコメント通知が表示されることを確認
3. `/settings/organizations` に遷移
   - 組織一覧 / メンバー / ロール（Owner / Admin / Member）を説明
   - 「DB レベルでは RLS + RBAC でアクセス制御している」ことを一言添える

話すポイント例:

- 「コメントやメンションは、アプリ内通知としてベルに集約されます」
- 「組織とロールの概念があり、Supabase の RLS でテナント分離を実現しています」

---

## 4. Analytics / Vitals / Settings（約 1 分）

1. `/app/analytics` を開く
   - 直近 30 日 / 8 週間のドキュメント数やユーザーアクティビティをざっと説明
2. `/app/vitals` を開く
   - Web Vitals のダッシュボード・エラー/稼働率のサンプル指標を見せる
3. `/settings/security` を開く
   - セキュリティチェックリストと `Security Design` へのリンクを見せる

話すポイント例:

- 「チーム利用を想定して、どれくらい使われているか・体験が劣化していないかを確認する簡易ダッシュボードを用意しています」
- 「セキュリティや運用ポリシーは docs/security.md / docs/operations.md に整理しており、プロダクトとして運用を意識した設計にしています」

---

## 5. What's New / Docs（約 30 秒）

1. `/app/whats-new` を開く
   - 直近の機能追加と「Coming Soon」を 1 行ずつ紹介
2. GitHub の `docs/` ディレクトリや Docsify ページに軽く触れる
   - `architecture`, `operations`, `security`, `case-study-startup` など

話すポイント例:

- 「What's New では、開発の進捗やロードマップ候補を簡単に追えるようにしています」
- 「ドキュメントは docs/ 以下にまとめており、アーキテクチャ / セキュリティ / 運用 / ケーススタディまで一通りそろえています」

---

この台本をベースに、実際の面接では「どこを深掘りしたいか」に応じて  
アーキテクチャ寄り / UX 寄り / チーム開発寄りに話を寄せていく想定です。
