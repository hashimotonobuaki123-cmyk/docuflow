## 🔐 Privacy & Data Policy - プライバシーとデータポリシー

DocuFlow で扱うデータの種類と、その取り扱い方針をまとめます。  
（実サービスでは利用規約 / プライバシーポリシーとして別途公開することを想定）

---

## 1. 取り扱うデータの種類

### 1.1 アカウント情報

- **メールアドレス**
  - 認証・パスワードリセット・（将来）通知に利用。
- **Supabase Auth のユーザーID (`auth.users.id`)**
  - アプリ内部では `user_id` として参照。

### 1.2 アプリケーションデータ

- **ドキュメント本体**
  - タイトル / カテゴリ / 本文（テキスト） / 要約 / タグ。
- **ファイル（PDF / Word）**
  - アップロード後にテキスト抽出し、本文として保存。
  - 現時点では Supabase Storage を使わず、テキストデータのみを保持する想定。
- **コメント**
  - ドキュメントに紐づくメモ・TODO・議論。
- **アクティビティログ**
  - 作成 / 更新 / 削除 / 共有などの操作履歴。

### 1.3 メタデータ

- 通知 (`notifications`)
  - 通知の種別、対象ドキュメント、既読状態。
- 組織・メンバー (`organizations`, `organization_members`)
  - チーム単位のワークスペース情報。

---

## 2. データの保存期間と削除

### 2.1 保存期間

- ドキュメント・コメント・アクティビティログ・通知:
  - 現時点では特に削除期限を設けず、**無期限で保存**。
  - 将来的に「○日以上前の通知は自動アーカイブ」などのポリシー追加を想定。

### 2.2 ユーザーによる削除

- ドキュメント削除:
  - `documents` 行を削除。
  - `document_versions` の履歴は残すかどうかは運用方針次第（現状は残す想定）。
- アカウント削除:
  - `SUPABASE_SERVICE_ROLE_KEY` が設定されている場合、  
    - Supabase Auth のユーザーを削除
    - 関連ドキュメント / バージョン / コメントも削除
  - 設定されていない場合は安全側に倒し、エラーメッセージを返す。

---

## 3. OpenAI へのデータ送信

### 3.1 送信内容

- ドキュメント本文（テキスト）を **プロンプトの一部として OpenAI API に送信**。
- タイトル・タグ・要約・埋め込みベクトル生成のために使用。

### 3.2 利用目的

- 仕様書・議事録・提案書などの要約・タグ付け・検索性向上。
- 類似ドキュメント検索（pgvector）用の埋め込み生成。

### 3.3 データ保持ポリシー（OpenAI 側）

- OpenAI の「データ保持・学習に関するポリシー」に準拠。
- 実サービス化する場合は、  
  - **ログ保存 OFF / モデル学習への利用 OFF** の API プランを選択する想定。

---

## 4. ログ・モニタリング

### 4.1 アプリケーションログ

- Vercel / Supabase / Sentry を利用し、以下を収集:
  - エラー内容
  - スタックトレース
  - 実行環境情報（ブラウザ / OS など）
- 個人を特定できる情報は極力含めない（メールアドレスなどはログに出さない方針）。

### 4.2 アクセスログ

- Supabase / Vercel のアクセスログを利用。
- IP アドレス・User-Agent 等は**運用上のトラブルシューティング目的**でのみ参照。

---

## 5. ユーザー権限と可視性

- **個人ワークスペース**
  - デフォルトでは自分のドキュメントは自分だけが閲覧・編集可能。
- **組織ワークスペース**
  - `organization_members` に所属しているメンバーのみが、その組織のドキュメントを閲覧・編集。
  - ロール（owner / admin / member）に応じて操作範囲を制限。
- **共有リンク**
  - URL を知っている人なら誰でも閲覧可能。
  - 編集・コメントは認証済みユーザーのみ。

---

## 6. ユーザーの権利（将来ポリシー案）

実サービス化を想定したときにユーザーに提供すべき権利の例:

- **アクセス権**: 自分のデータをエクスポートする権利（例: JSON/CSV）。
- **削除権**: 自分のアカウントと関連データを削除する権利。
- **訂正権**: メールアドレスや表示名などの修正手段。

現時点では開発用プロジェクトであり、上記は **将来の正式なポリシー策定に向けた草案** として位置づける。
