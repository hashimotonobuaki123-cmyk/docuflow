## 🌏 i18n & Accessibility Design - 多言語対応とアクセシビリティ方針

DocuFlow を「日本語中心の SaaS」から、将来的に英語圏・グローバルでも使えるようにするための設計メモです。

---

## 1. 多言語対応 (i18n) 方針

### 1.1 現状

- UI テキストは基本的に **日本語ベース**。
- README / ランディングページには **英語サマリー** を併記。

### 1.2 方針

- Next.js 公式 i18n もしくは `next-intl` を用いて、`ja` / `en` の 2 言語対応を目指す。
- 優先度は以下の順:
  1. `/`（ランディング）と `/app`（ダッシュボード）の主要文言
  2. 認証画面（login / signup）
  3. 設定画面

### 1.3 実装イメージ（将来）

```tsx
// app/[locale]/layout.tsx
export async function generateStaticParams() {
  return [{ locale: "ja" }, { locale: "en" }];
}

export default function RootLayout({
  params: { locale },
  children,
}: {
  params: { locale: "ja" | "en" };
  children: React.ReactNode;
}) {
  return (
    <html lang={locale}>
      <body>{children}</body>
    </html>
  );
}
```

## 2. アクセシビリティ (a11y) 方針

### 2.1 スクリーンリーダー対応

- 主要ボタン（検索、作成、新規作成、削除）には `aria-label` を追加。
- アイコンのみのボタン（ピン留め、お気に入り、削除、通知ベルなど）は、  
  視覚的なアイコンとは別に **音声読み上げテキスト** を持つようにする。

### 2.2 キーボード操作

- すでに実装済み:
  - `Cmd+K` でコマンドパレット
  - `/` で検索フォーカス
- 今後の改善:
  - モーダル・ダイアログで `Esc` で閉じる
  - `Tab` 移動の順序（フォーカストラップ）を整理

### 2.3 コントラストとテーマ

- ダーク / ライトテーマをサポートしつつ、WCAG AA レベルのコントラストを目標とする。
- Tailwind のカラートークンを使い、極端に薄い色（`slate-100` など）がテキストに使われないようにする。
